# docs/

Project documentation — requirements, architecture, and decision records.

## Subfolders

| Folder | Purpose |
|---|---|
| `PRD/` | Product Requirements Document (v2.0) — functional requirements, personas, business model |
| `TRD/` | Technical Requirements Document (v2.0) — system architecture, data architecture, API design |
| `ROADMAP/` | Engineering Roadmap (v2.0) — Track 1 (Engineering), Track 2 (Research), Track 3 (Pilot/GTM) |
| `ADRs/` | Architecture Decision Records — rationale for key technical decisions |
| `architecture/` | System diagrams, data flow charts, deployment topology |

## Key Documents

- **PRD v2.0** — Source of truth for product scope and requirements
- **TRD v2.0** — Source of truth for technical architecture and implementation
- **ROADMAP v2.0** — Source of truth for epic sequencing and dependencies
- **DATA_AND_MODEL_STRATEGY** — ML/data engineering reference for feature engineering, model strategy, labeling
- **REPOSITORY_STRUCTURE_TRACK1** — Canonical repository structure for Track 1 epics
